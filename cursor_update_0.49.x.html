Cursor 0.49.x 更新日志

2025年4月15日

主要更新

  • 规则生成：现在可以直接在对话中使用`/Generate Cursor Rules`命令来生成规则。
  • 自动附加规则：对于设置了路径模式的自动附加规则，代理现在会在读取或写入文件时自动应用正确的规则。
  • 规则编辑可靠性：提高了规则编辑的可靠性。

更易访问的历史记录

聊天历史已移至命令面板中，可以通过聊天中的“显示历史记录”按钮或通过“显示聊天历史记录”命令访问。

简化代码审查

在每次对话结束时提供内置的差异视图，简化了对代理代码的审查。你可以在对话末尾找到“审查更改”按钮。

MCP中的图像支持

可以将图像作为MCP服务器上下文的一部分传递。当截图、UI模型或图表为问题或提示添加重要背景时，这一点非常有用。

改进的代理终端控制

增加了对代理启动的终端工作流的更多控制。现在可以在运行之前编辑命令，或者完全跳过它们。同时，“弹出”选项被重命名为“移到后台”，以更好地反映其功能。

全局忽略文件

可以通过用户级设置定义跨所有项目的忽略模式。这可以防止构建输出或机密等嘈杂或敏感文件出现在提示中，而无需每个项目进行配置。

新增模型

  • Gemini 2.5 Pro
  • Grok 3
  • Grok 3 Mini
  • GPT-4.1
  • o3
  • o4-mini

项目结构上下文（测试版）

引入了一个选项，可以在上下文中包含项目结构，从而将目录结构添加到提示中。代理现在能够更清楚地了解你的项目组织方式，从而改进大型或嵌套单体仓库的建议和导航。

键绑定

  • 某些CMD+K快捷键现在可重新映射
  • Emacs键绑定扩展现在可靠工作

改进

  • 自动选择的新模型选择器UI
  • 命令面板的新界面
  • Tab跳转建议的UI刷新。视口外的建议现在可点击
  • 聊天中的模式工具提示
  • MCP稳定性提升：现在可以使用访问密钥和秘密密钥连接到AWS Bedrock

修复

  • 始终附加的规则现在在总结过程中正确持续
  • .mdc文件中的规则现在可以由代理创建和编辑，不会出现问题
  • 修复了输入框开头@提及节点的选择问题
  • 核心编辑器性能

业务/团队

  • Cursor现在可以向上遍历目录树以查找忽略文件。此行为默认关闭，可以从管理设置中启用。
  • 企业现在可以使用IAM角色连接到AWS Bedrock。
  • 管理员现在可以直接从仪表板查看每位用户的支出和快速请求使用情况。
  • 管理员可以从仪表板配置全局MCP设置。

Major Updates

  • Rule Generation: You can now generate rules directly in the conversation using the `/Generate Cursor Rules` command.
  • Auto-attach Rules: For auto-attach rules with path patterns, the agent will now automatically apply the correct rules when reading or writing files.
  • Rule Editing Reliability: Improved the reliability of rule editing.

More Accessible History

Chat history has been moved to the command palette and can be accessed via the "Show History" button in the chat or through the "Show Chat History" command.

Simplified Code Review

A built-in diff view is provided at the end of each conversation, simplifying the review of agent code. You can find the "Review Changes" button at the end of the conversation.

Image Support in MCP

Images can now be passed as part of the MCP server context. This is particularly useful when screenshots, UI mockups, or diagrams add important context to a question or prompt.

Improved Agent Terminal Control

Added more control over terminal workflows initiated by the agent. Commands can now be edited before running or skipped entirely. Additionally, the "Pop Out" option has been renamed to "Move to Background" to better reflect its functionality.

Global Ignore Files

You can now define ignore patterns across all projects via user-level settings. This prevents noisy or sensitive files like build outputs or secrets from appearing in prompts without requiring per-project configuration.

New Models

  • Gemini 2.5 Pro
  • Grok 3
  • Grok 3 Mini
  • GPT-4.1
  • o3
  • o4-mini

Project Structure Context (Beta)

Introduced an option to include project structure in the context, adding the directory structure to the prompt. Agents can now better understand your project organization, improving suggestions and navigation for large or nested monorepos.

Key Bindings

  • Some CMD+K shortcuts can now be remapped
  • Emacs key binding extensions now work reliably

Improvements

  • Auto-selected new model selector UI
  • New interface for the command palette
  • UI refresh for Tab jump suggestions. Suggestions outside the viewport are now clickable
  • Mode tooltips in chat
  • MCP stability improvements: Now supports connecting to AWS Bedrock using access keys and secret keys

Fixes

  • Always-attached rules now persist correctly during summarization
  • Rules in .mdc files can now be created and edited by agents without issues
  • Fixed selection issues with @mention nodes at the beginning of the input box
  • Core editor performance

Business/Team

  • Cursor can now traverse up the directory tree to find ignore files. This behavior is off by default and can be enabled from the admin settings.
  • Enterprises can now use IAM roles to connect to AWS Bedrock.
  • Admins can now view each user's spending and quick request usage directly from the dashboard.
  • Admins can configure global MCP settings from the dashboard.
